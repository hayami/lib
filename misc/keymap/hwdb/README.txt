/etc/udev/hwdb.d に設定ファイルを追加・削除・変更した時は、以下のコマンドで
その設定内容を反映させることができます。

        $ sudo systemd-hwdb update
        $ sudo udevadm trigger (--subsystem-match=input ...)

パフォーマンス上の注意
    udevadm trigger を実行する際は、--subsystem-match=... で、その対象を絞り
    込んだほうがよいらしい。

キーマップを削除する場合
    udevadm trigger は追加・変更されたキーマッピングだけをカーネルに反映しま
    す。既にカーネルに読み込まれたマッピングは、設定ファイルからマッピングを
    削除し、上記コマンドを実行しても、そのマッピングは残ったままになります。
    そもそも hwdb のキーマッピングは、udev の keyboard ビルトインが ioctl で
    カーネル内のマッピングを書き換える仕組みで、いったん書き換わった後は、明
    示的に上書きしない限り元に戻りません。

    削除を反映したい場合は、元のキーマップに戻す行 (デフォルト値を指定する行)
    を書いて上記コマンドを実行する必要があります。
